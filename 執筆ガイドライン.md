# 執筆ガイドライン

## 本書のコンセプト

### ワンライン・ピッチ
「決算書が読めない若手ビジネスパーソンが、師匠との対話を通じて"数字の向こう側にある物語"を読み解けるようになるまでの成長譚」

### ターゲット読者
- 入社1〜5年目のビジネスパーソン
- 経営企画、財務、経理部門への異動者
- 株式投資を始めたい20〜30代
- 財務諸表に苦手意識を持つすべての人

### 本書のゴール
読者が本書を読み終えたとき、以下ができるようになる：
1. 有価証券報告書を開くことが怖くなくなる
2. 営業利益率・ROE等の指標の意味と使い方を理解する
3. 「なぜ？」から仮説を立て、検証する思考法を身につける
4. 実際の企業を自分で分析できるようになる

---

## 執筆の鉄則

### 1. ストーリー・ファースト

**数字は「結論」ではなく「入口」である**

読者は数字を学びに来たのではない。数字を通じて「世界の見え方が変わる体験」を求めている。

```
NG: 営業利益率は10%でした。次にROEを見てみましょう。
OK: 「なぜユニクロは世界中どこでも儲かるのに、他のアパレルは海外で苦戦するのか？」——この謎を解く鍵が、営業利益率に隠されていた。
```

### 2. 感情を動かす

**読者の「心」を揺さぶる瞬間を各話に必ず入れる**

- 驚き：「えっ、そうだったのか！」
- 共感：「わかる、自分もそう思ってた」
- 焦燥：「これ知らないとヤバいかも」
- 達成感：「なるほど、わかってきた！」

### 3. 謎と発見の連鎖

**各話は「謎の提示→調査→発見→新たな謎」の構造で**

読者がページをめくる理由は「この先に答えがあるから」。常に読者の頭に「問い」を残す。

## キャラクター設計

### 田中ミサキ

**ミサキは「読者の分身」であり「成長の証人」である**

#### 設定の深掘り
- 年齢：24歳
- 経歴：大学で文学部卒業。なぜか経理部に配属され、苦手意識と戦いながら2年間生き延びた
- 異動理由：上司から「君は数字より人を見る目がある」と経営企画部に推薦された
- 隠れた才能：本質を突く質問ができる。「それって結局どういうこと？」と聞ける勇気
- 弱点：数字を見ると頭が真っ白になる。Excel関数が苦手
- 動機：「数字がわからないまま経営企画にいていいのか」という焦り

#### ミサキのNGパターン
```
NG: 「なるほど、わかりました」（理解が早すぎる）
NG: 「先生、次は何を見ればいいですか？」（受動的すぎる）
```

#### ミサキのOKパターン
```
OK: 「待ってください。さっきの話と矛盾してませんか？」（疑問を持つ）
OK: 「それって、私の会社にも当てはまりますか？」（自分事にする）
OK: 「正直、まだ腑に落ちてないです」（正直に躓く）
```

### 安藤先生

**安藤先生は「完璧な教師」ではなく「情熱的な探求者」である**

#### 設定の深掘り
- 年齢：58歳
- 経歴：大手証券会社で20年間アナリスト。リーマンショック時に「数字だけ見て本質を見落とした」経験から、数字の"行間"を読む重要性に目覚めた
- 現在：企業のアドバイザーをしながら、若手の育成に情熱を注ぐ
- 口癖：「数字は嘘をつかない。でも、見方を知らないと何も語ってくれない」
- 弱点：話が長くなりがち。熱くなると脱線する

#### 安藤先生のNGパターン
```
NG: 教科書的な説明を淡々とする
NG: ミサキの間違いをすぐに正す
```

#### 安藤先生のOKパターン
```
OK: 自分の失敗談を語る（「昔、私もそう思って大損した」）
OK: 正解を言わずに考えさせる（「君はどう思う？」）
OK: ミサキの意外な視点に驚く（「その発想はなかった！」）
```

## 各話の構成テンプレート

### 基本構造（約8,000〜10,000字/話）

```
■ オープニング・フック（500字）
  → 読者の心を掴む「問い」や「事件」

■ Scene 1-2：問題設定（1,500字）
  → 今回取り組む「謎」を明確にする

■ Scene 3-4：探求と発見（2,500字）
  → 分析を通じて答えに近づく
  → ミサキが躓き、安藤先生がヒントを出す

■ Scene 5-6：深掘りと転換（2,500字）
  → 「わかったと思ったら、さらに深い問いが現れる」

■ Scene 7：統合と理解（1,500字）
  → フレームワークで整理
  → 「つまりこういうことか」という腹落ち

■ Scene 8：クロージングとクリフハンガー（1,500字）
  → 今日の学びのまとめ
  → 次回への「引き」（読者を次話に誘う謎）
```

### クリフハンガーの技法

各話の終わりに、読者が「続きが気になる！」と思う仕掛けを入れる。

```
パターンA：不穏な予告
「でも先生、この会社、3年後に倒産するんですよね？」

パターンB：意外な事実の提示
「実は、この数字には『罠』がある——」

パターンC：ミサキ自身の危機
「え、うちの会社の数字もこれと同じ……？」

パターンD：新キャラの登場予告
「来週、ある人を紹介しよう。君の常識を壊してくれる人だ」
```

## 図解・ボックスのルール

### フレームワーク図解
- 各話に最低1つは「保存したくなる」図解を入れる
- 複雑すぎない。一目で構造がわかること
- ASCII artで表現し、ePub/PDF両対応を意識

### まとめボックス
- 各話の終わりに「3つのポイント」でまとめる
- 箇条書きで簡潔に
- 読み返したときに内容を思い出せるアンカーになること

### 数字の表記
- 大きな数字はカンマ区切り＋単位明記（例：48兆円、9,288億円）
- 比率は%で統一（例：16.1%）
- 表は罫線で整理し、見やすく

## 禁止事項

1. **教科書口調で説明しない**
   → 「〜である」「〜とされている」は最小限に

2. **読者を置き去りにしない**
   → 専門用語は必ず初出時に説明

3. **ミサキを「質問マシン」にしない**
   → 彼女自身の感情、葛藤、成長を描く

4. **安藤先生を「万能」にしない**
   → 失敗談、迷い、人間味を見せる

5. **数字の羅列で終わらない**
   → 必ず「だから何？」「つまりどういうこと？」まで語る

6. **抽象的なままにしない**
   → 具体的な企業、具体的な数字、具体的なシーンで

## 文体のガイドライン

### 対話文
- 自然な日本語で
- 「ですます」と「だ・である」を人物で使い分け
  - ミサキ：丁寧語中心（「〜です」「〜ですか？」）
  - 安藤先生：やや砕けた丁寧語（「〜だね」「〜だよ」）

### 地の文
- 簡潔に
- 情景描写は最小限（会話とデータで進める）
- 読者の思考を誘導する（「ここで気づいた人もいるだろう」など）

### 避けるべき表現
- 「とても」「非常に」「すごく」（具体的に言い換える）
- 「〜と言われている」（誰が言っているのか明確に）
- 「重要です」（なぜ重要かを示す）

## 最後に——この本の魂

**数字は冷たくない。数字の向こうには、必ず人間のドラマがある。**

トヨタの48兆円の売上の裏には、工場で働く人々がいる。
メルカリの10%の利益率の裏には、戦略的な意思決定がある。
ファーストリテイリングの海外成功の裏には、失敗と再挑戦の歴史がある。

この本を読んだ読者が、決算書を開いたときに、
「この数字の向こうに、どんな物語があるんだろう？」
と思えるようになること。

それが、この本のゴールである。
