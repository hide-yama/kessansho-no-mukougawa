# 会社の数字の読み方 プロジェクト

## プロジェクト概要

**書籍タイトル**: 決算書の向こう側 〜プロが教える13の解読術〜

**ジャンル**: ビジネス・エンターテイメント（ストーリー形式の決算書入門）

**コンセプト**: 数字が苦手な若手社員が、伝説のアナリストとの出会いを通じて決算書を読み解く力を身につける13週間の成長物語

**コアメッセージ**: 「数字は嘘をつかない。でも、見方を知らないと何も語ってくれない」

---

## 開発プロセス：探求型執筆

### この書籍開発は「研究」である

本プロジェクトは単なる知識本・レクチャー本の制作ではない。
**書籍開発そのものを研究論文の制作行為として進める。**

```
┌─────────────────────────────────────────────────┐
│           探求型執筆サイクル                     │
├─────────────────────────────────────────────────┤
│                                                 │
│   執筆 ──→ 仮説発生 ──→ リサーチ依頼           │
│     ↑                        ↓                 │
│     │                   ユーザーが調査          │
│     │                        ↓                 │
│     └──── 新事実発見 ←── research/に格納       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### AIからのリサーチ依頼ルール

執筆中に以下の状況が発生した場合、**AIからユーザーにリサーチを依頼する**：

1. **仮説が生まれた時**: 「この展開には〇〇のデータが必要では？」
2. **矛盾を発見した時**: 「この数字の整合性を確認したい」
3. **深掘りが必要な時**: 「この企業の〇〇についてもっと知りたい」
4. **新しい比較軸を思いついた時**: 「〇〇と△△を比較すると面白いのでは？」

**依頼フォーマット**:
```
【リサーチ依頼】
■ 背景: なぜこの情報が必要か
■ 調査対象: 具体的に何を調べるか
■ 期待する成果物: どんな形式で欲しいか
■ 格納先: research/のどこに置くか
```

### ユーザーからの資料提供

ユーザーが `research/` フォルダに新しい資料を格納した場合：
- その内容をストーリー展開に積極的に活かす
- 新事実によって物語がより深く、リアルになることを歓迎する
- 必要に応じて既存の設定を更新する

## AIとの協働ルール

### 制作ノート（`制作ノート.md`）の役割

AIとの会話はセッション切断で消えるため、**制作ノートに進捗と決定事項を記録**する。

**目的**：
- セッションが切れても文脈を復元できる
- 決定事項を後から参照できる
- 進捗状況を可視化する

**セッション復帰時の手順**：
1. `CLAUDE.md` を読んでプロジェクト概要を把握
2. `制作ノート.md` を読んで直近の会話と進捗を確認
3. 「ここから続けてください」で再開

### 制作ノートの書式

会話ログは以下の形式で記録：

```markdown
---

**[私]** 2026-01-13

（ユーザーの発言）

---

**[AI]** 2026-01-13

（AIの返答）

---
```

- `---`（区切り線）で会話を分離
- `**[私]**` `**[AI]**` で話者を明示
- 日付を入れると後から追いやすい
- 見出し（#）は使わない（ファイル構造と衝突するため）

### 進捗の記録ルール

AIは作業完了後、必ず `制作ノート.md` に以下を追記する：
- 実行した作業内容
- 作成・更新したファイル
- 現在のステータス（Phase、完了タスク）
- 次のアクション

## 書籍構成

- **全13話 + 付録**
- **各話**: 約10,000〜12,000字 + コラム（800〜1,200字）
- **読了時間**: 約25〜30分/話（全体で約7〜8時間）

### 三部構成

| 部 | 話数 | テーマ |
|---|---|---|
| 第I部：覚醒編 | 1〜5話 | 営業利益率、セグメント、CF、B/S、銀行財務 |
| 第II部：探求編 | 6〜9話 | ROE/ROA、PER/PBR、成長企業、成熟企業 |
| 第III部：実践編 | 10〜13話 | 競合分析、業界分析、統合分析、行動へ |

## 物語の方針：後半の展開（検討中）

### クライマックス：半沢直樹展開（採用）

**第11〜13話で、専務の不正をミサキが暴き、土下座させる展開。**

#### 敵役（案）
**黒田専務（60歳）**
- 東洋テック専務取締役、海外事業本部長
- 次期社長の最有力候補
- 「海外事業を立て直した功労者」として社内評価が高い
- 実態：数字を操作して自分の手柄を作っていた

#### 不正の内容（財務分析で発見可能なもの）
| 手口 | 財務上の兆候 | 学べるスキル |
|------|-------------|-------------|
| 架空売上の計上 | 売掛金の異常な膨張 | 売上債権回転期間の分析 |
| 在庫の水増し | 棚卸資産の急増 | 在庫回転期間の分析 |
| 費用の先送り | 利益とCFの乖離 | CF計算書の読み方 |
| 関連会社への付け替え | 関連当事者取引の急増 | 有報の注記の読み方 |

#### 展開イメージ
```
第11話：不穏な数字を発見（売掛金とCFの異常）
第12話：証拠を固める（関連当事者取引の調査）
第13話：経営会議で告発 → 専務土下座
```

**※この展開は1話から書き進めながら詳細化する。執筆中に新たな仮説やアイデアが生まれた場合は随時更新。**

## 執筆ルール（重要）

### 数字の表記
- 大きな数字：カンマ区切り＋単位（例：48兆円、9,288億円）
- 比率：%統一（例：16.1%）
- 小数点：第1位まで（例：10.2%）

### 禁止表現
| 禁止 | 代替 |
|-----|-----|
| とても/非常に/すごく | 具体的な数字や比較で表現 |
| 〜と言われている | 誰が言っているか明記 |
| 重要です | なぜ重要かを示す |

### 対話文
- **ミサキ**: 丁寧語、疑問形が多い、心の声を（　）で描写
- **安藤先生**: やや砕けた丁寧語、問いかけが多い
